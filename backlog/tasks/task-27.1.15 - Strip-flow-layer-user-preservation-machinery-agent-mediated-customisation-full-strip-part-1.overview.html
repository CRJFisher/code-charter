The full strip: removing a safety net for a state that cannot occur

A code-charter diagram is rebuilt from the code by an agent. At the flow layer, every node description is agent-generated; higher-layer chart customisation is agent-mediated — the agent authors it and re-applies it on each sync. No diagram layer holds direct human byte-edits that an agentic pass must avoid overwriting. So the user-preservation and recovery machinery built across task 27.1 — the user-tier writes, the overlap remap, the re-attachment bin, the recovery MCP tools, the relocation accept-dance — protected a state that never happens. The full strip removed it. Part 1 and all six follow-up sub-tasks are now done and merged.

part 1 done & merged 6 / 6 follow-ups done labels: drift · flows · graph-db · simplification
The premise in one line Nothing on the diagram is a protected human edit; everything on it is either re-derived from the code each sync or re-authored by the agent. Therefore the preservation apparatus was surplus — and per the no-surplus-code principle, it is gone.

What part 1 did, and what the batch finished

Part 1 (this task, merged) stripped the flow-layer half: no more user-tier writes, the ≥50% Jaccard overlap remap deleted, the write-only anchor_set_hash gone, the re-attachment bin and its drift.list/drift.next recovery tools removed, and write_descriptions turned into an unconditional agentic upsert that resurrects a stranded description instead of leaving it “for drift.resolve”. The strip-map companion shows exactly what was removed versus kept.

Part 1 deliberately left three things for the follow-up batch, all now shipped: the core relocation accept-dance (kept so its blast radius could be reviewed on its own — now an inline re-anchor), one uncovered retirement case (demotion-by-wrapper — now closed with demoted-seed + subsumption), and the design docs that described the deleted surface (now amended). The batch also added the safety and observability the retirement path needed and removed a now-orphaned storage API.
