Strip options for the flow-layer simplification

At the flow layer (auto-detected call-graph flows, task 27.1), node descriptions are agent-generated at flow-creation time only. There are no user-customisable elements here — all user customisation moves up to a later flow-diagrams layer that does not yet exist. A large band of machinery built across task 27.1 exists solely to preserve and recover user-authored content. With no user content at this layer, much of it is dead. Three reviewers agreed on a core to remove and diverged on how far to push the recovery surface. Pick one depth below.

3 depths scope packages/drift + packages/core recommended Medium 1 shared footgun

Why a strip exists at all

The flow layer never sees user edits. Every code path that asks "is this field user-owned, and must I preserve or recover it?" now guards a state nothing at this layer can produce. The reviewers split the answer into three nested depths: each deeper depth removes everything the shallower one does, plus more of the recovery product surface. The bands in the comparison below are cumulative — Medium is Minimal plus its own additions, Full is Medium plus its own.

The proving fact (true in all three depths)

The only call that ever stamps a field as user-owned in the whole flow layer is restamp_carried_label. Every layer === "user" guard downstream defends a state that nothing else can now write. Remove the writer and the guards become dead by construction.

packages/drift/src/reconcile/hydrate.ts · 108–113
/** Re-stamp a carried user label as user-owned so a later agentic pass cannot overwrite it. */
function restamp_carried_label(deps, flow_id, remap) {
  if (remap?.label !== undefined) {
    deps.store.write_fields({ kind: "node", id: flow_id }, { label: remap.label }, "user");  // ← the ONLY write_fields(…,"user") at this layer
  }
}

The shared footgun

This applies to Medium and Full — any depth that removes the re-attachment bin or drift.resolve. Description writing currently skips writing onto a binned (soft-deleted) side-node, leaving the regeneration to drift.resolve. Remove the bin without touching this line and a stranded agentic description is never regenerated — lost forever.

Do-not-cut rail — looks dead, is load-bearing

These four survive in every depth. Each looks like recovery machinery but earns its keep for an unrelated reason. The first is the sharpest trap: it shares a name prefix with a cut item.

Keep in all three depths
  • anchor_set (the ARRAY, not the _hash) — drives membership-drift re-sync in affected_flows.ts:55 (const stored = flow.node.attributes.anchor_set). The trap: it shares a prefix with the dead anchor_set_hash. Cut the hash; keep the array.
  • the content-hash re-describe SKIP (describe.ts existing_descriptions / the cached bucket) — a cost guard that avoids re-paying the LLM for unchanged members. Not user preservation.
  • core user tier + field-ladder + resolver + re_extract reconcile — foundational task-27.0 primitives the deferred higher layer will need.
  • soft-delete as a RETIREMENT mechanism — retiring a superseded flow so render shows no dead diagram. Independent of recovery; keep it.

Why Medium is recommended Minimal removes the dead guards but leaves a whole recovery product (bin, MCP tools, SessionStart banner) running with nothing left to recover — clutter that keeps printing at session start. Full reaches into packages/core reextract semantics, which is a large enough change to be its own task. Medium removes the pointless product surface while keeping the core relocation/reanchor primitives that task-27.0 still relies on — paying the footgun fix once, in the same change.

Impact on already-shipped work

Watch — sunk cost Tasks 27.1.6.2 and 27.1.6.3 were entirely about the re-attachment bin: the mass-bin fix, then the richer recovery with ranked candidate targets and drift.next. Medium and Full substantially undo that shipped work. Minimal leaves it standing. If undoing recently-shipped recovery features is a hard blocker, that argues for Minimal; if the goal is a clean flow layer with no orphaned product surface, Medium is the call and the prior work is accepted as scaffolding that did its job during the transition.
