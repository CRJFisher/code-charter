Why this exists

Task-27.1.6 turns the inert substrate the earlier subtasks built into a live loop. Before it, no production path populates the on-disk graph store and the drift-sync skill is a no-op stub — so a hydrated flow can never reach the webview. This task wires the full Stop-hook → sub-agent → skill → reconcile engine → store path end to end, doing double duty: HYDRATE a flow on first work, RE-SYNC it on later edits — always updating, never gating, always re-applying your authored description / name / pin.

The loop the parent ships

One Stop-hook entry point, two branches, with preservation woven through both. A genuine miss — content whose anchor no longer resolves, or a split/merge that strands a name — goes to the re-attachment bin, recoverable and never auto-pruned.

No review apparatus in v1

No observation/adjudication tables, no open→triaged→resolved lifecycle, no cosmetic/intent classifier, no PreCommit gate, no drift inbox. The diagram absorbs drift out-of-band, off your attention. Those are deferred to task-27.1.9 / task-27.1.10.

How the sub-tasks fit together

With the parent Done, the children are the come-back-and-make-it-real pass: prove and sand the recovery surface, fix a re-sync bug the live loop surfaced, then extend recovery and reach beyond code into standalone docs.

Stop only for flow-relevant edits

Pre-filter worked_on so a doc/config-only turn does not trigger a no-op full-repo reconcile. Layers on the per-turn watermark already shipped; resolves the standalone-doc question (ignore-and-log).

task-27.1.6.2

Fix the re-sync mass-bin bug

A separate defect the live loop exposed: two anonymous callbacks in one file collide in symbol_path space, build_index empties the resolver index on the duplicate, and re-sync then mass-bins every preserved description (25 of 29 in the dogfood run). Skip <anonymous> at the source; dedup before building as defense-in-depth.

4
task-27.1.6.3

Richer drift recovery

The kept drift.* surface covers two strandings — commit a relocation, or restore onto the original anchor — but not the middle case: the symbol is gone and you know the right new one. Four additions reach it: reattach a binned description onto a chosen current symbol (carrying authored fields through reanchor_node), ranked candidates[] computed store-only from live anchored nodes, a kind disambiguator that stops node-vs-edge guessing, and a resolve-next loop primitive.

5
task-27.1.6.4

Standalone-doc flows

Doc nodes and doc↔code linkage so unconnected .md edits can be reconciled — extends the loop past code-only flows into documentation.
