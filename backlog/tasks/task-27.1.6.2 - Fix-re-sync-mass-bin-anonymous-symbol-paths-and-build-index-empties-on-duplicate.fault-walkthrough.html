Two anonymous callbacks in one file empty the resolver index — and re-sync bins every preserved description

A live re-sync of a real repo soft-deletes valid, current agentic.description side-nodes into the re-attachment bin. In the dogfood run that surfaced it, 25 of 29 descriptions were binned in a single re-sync of unchanged-meaning code. The fault is a two-step chain, both halves in the code→diagram preservation path.

The blast radius

The data is recoverable — it lands in the bin, not deleted — but a re-sync must never strand content whose anchor still resolves. One pathological file silently empties the index for the whole changed set, so every preserved, anchored description in that re-sync misses and is binned. This is the unimplemented half of task-27.1.6's review finding C.

The two-hop failure chain

The trigger is mundane: a file with two or more bodied anonymous callbacks. Ariadne names each one <anonymous>, so they derive the same symbol_path. From there a single duplicate cascades, through a misrouted catch, into a mass soft-delete.

the actual fault path (rust) the outcome the fix restores (olive, dashed)
1

Two bodied anonymous callbacks share one symbol_path

Ariadne gives an unnamed callable name === "<anonymous>". walk_callables emits one record per bodied callable including these, so a file with ≥2 anon callbacks derives two identical <file>#<anonymous>:function paths. An unnamed callback has no stable, addressable identity — it should not be a resolver symbol at all.

packages/core/src/resolver/from_ariadne.ts :124–154 (walk_callables), :157, :193 (the two emitters)

2

build_resolver_index throws on the duplicate

The index requires symbol_path to be unique. Two indistinguishable symbols are treated as a derivation defect to fix upstream — never silently overwritten, because that would make the hit / body-changed arms depend on input order. So it throws duplicate symbol_path in resolver index.

packages/core/src/resolver/resolver_index.ts :24–49 (throw at :30–32)

3

build_index's outer catch swallows it into an empty index

The comment claims build_index isolates a thrower per file. It does not. The inner try wraps resolver_symbols_from_ariadne, which never throws; the real throw — build_resolver_index(symbols) — sits in the outer try, whose catch returns build_resolver_index([]). One duplicate anywhere in the set collapses the whole index to empty.

packages/drift/src/reconcile/ariadne_adapter.ts :72–89 (build_index)

4

re_extract resolves every preserved description against the empty index

Against an empty index every anchored node returns miss. A miss means the symbol is both renamed and re-bodied, so reconcile_node calls store.soft_delete(target) — moving the description into the re-attachment bin. With the index empty, that happens to the entire changed set at once.

The misleading try/catch in build_index today

The whole defect lives in which try the throwing call sits in. The annotations mark exactly where the throw lands and where it gets swallowed.

packages/drift/src/reconcile/ariadne_adapter.ts — build_index (current, faulty)
build_index(file_set) {
  // `build_resolver_index` throws on a duplicate symbol_path (a derivation defect in one
  // file); a single pathological file must not break the hook, so index file-by-file
  // and skip a thrower.   ← the comment describes a per-file skip that never happens
  const symbols = [];
  for (const input of file_inputs(project, file_set)) {
    try {                                                       ← INNER try
      symbols.push(...resolver_symbols_from_ariadne([input]));   ← this NEVER throws
    } catch {
      // skip the offending file's symbols; the rest of the set still reconciles.
    }
  }
  try {                                                         ← OUTER try
    return build_resolver_index(symbols);                      ← the REAL throw lands here
  } catch {
    // A cross-file duplicate: fall back to an empty index rather than aborting.
    return build_resolver_index([]);                          ← swallowed → EMPTY index for the whole set
  }
}
Why the tests shipped green

The committed build_index describes a per-file skip in its comment but never does it, and the suite never exercised a file with multiple anonymous callbacks. No duplicate was ever produced in a test, so the outer catch never fired and the empty-index fallback was never observed.

The two-part fix — and why both halves are needed

Skip-at-source removes the duplicates entirely; dedup-in-build_index is defense-in-depth so a residual duplicate can never empty the index again. Either half alone leaves a hole.

Part 1 — skip the <anonymous> sentinel in walk_callables

One guard in the shared traversal removes anonymous callables from both resolver_symbols_from_ariadne and anchored_symbols_from_ariadne at the source — no colliding paths, and no pointless <anonymous> descriptions for unaddressable callbacks.

packages/core/src/resolver/from_ariadne.ts — walk_callables (proposed)
function walk_callables(file, visit) {
  const visit_named = (def, enclosing) => {
    // Anonymous callables have no rename-stable identity and collide in symbol_path
    // space; they are never anchorable resolver symbols.
    if (def.name === "<anonymous>") return;
    visit(def, enclosing);
  };
  for (const def of file.definitions) {
    // … each arm now calls visit_named(...) instead of visit(...)
  }
}

Confirm the exact sentinel Ariadne emits before pinning the literal. The diagnostic shows <anonymous>.

Part 2 — dedup by symbol_path before build_resolver_index

Even with Part 1, a future derivation defect could re-introduce a duplicate. Dedup the accumulated ResolverSymbol[] first-wins, deterministic, before building — so a duplicate can never reach the throwing path and empty the index. Log any drop (AC#5: no silent narrowing). The misleading per-file try/catch is removed.

packages/drift/src/reconcile/ariadne_adapter.ts — build_index (proposed)
build_index(file_set) {
  const symbols = [];
  for (const input of file_inputs(project, file_set)) {
    try { symbols.push(...resolver_symbols_from_ariadne([input])); } catch {}
  }
  try { return build_resolver_index(symbols); }
  catch { return build_resolver_index([]); }
  const symbols = resolver_symbols_from_ariadne(file_inputs(project, file_set));
  const seen = new Set<string>();
  const deduped = [];
  for (const sym of symbols) {
    const path = derive_symbol_path(sym);          // first-wins, deterministic order
    if (seen.has(path)) { log_drop(path); continue; }  // AC#5: report, never silently cap
    seen.add(path); deduped.push(sym);
  }
  return build_resolver_index(deduped);          // can no longer throw on a duplicate
}
Why both, not one

Part 1 alone stops this collision but leaves build_index still able to wipe the index if any other source of duplicates appears — the mass-bin remains one derivation bug away. Part 2 alone keeps the index populated but still emits meaningless <anonymous> descriptions for unaddressable callbacks, and arbitrarily drops one of two real anon symbols. Together: no anonymous symbols are ever produced, and a residual duplicate can never empty the index again.

Acceptance criteria

The five conditions that close the bug. The regression test (AC#4) is the safeguard that the green-test gap can never recur.

  • #1 No anonymous resolver symbols. Both emitters skip callables with no stable name (name === "<anonymous>"), so they never emit colliding <file>#<anonymous>:… records and never produce descriptions for unaddressable callbacks.
  • #2 build_index never empties on a duplicate. A residual duplicate anywhere in the file set must not collapse the index to empty — dedup by symbol_path (deterministic, first-wins) before build_resolver_index, so every non-colliding symbol still resolves.
  • #3 Re-sync preserves resolvable descriptions. Re-syncing a file with multiple anonymous callbacks does not soft-delete any agentic.description whose symbol_path:content_hash still resolves; only genuine relocations / misses are staged or binned.
  • #4 Regression test. A fixture with ≥2 bodied anonymous functions plus a named, described symbol drives a re-sync and asserts (a) build_index returns a populated index, and (b) the named symbol's description survives (not binned).
  • #5 No silent narrowing. If dedup drops a duplicate it is logged / reported (never a silent cap), consistent with the rest of the engine.
Out of scope — do not conflate

The call-graph layer legitimately includes <anonymous> entry points, so flow detection still emits small …#<anonymous>:function flows — deterministic v1 grouping noise, the agent-judged umbrella concern (task-27.1.7), separate from this resolver / preservation fix. The Stop-hook trigger UX is task-27.1.6.1.
