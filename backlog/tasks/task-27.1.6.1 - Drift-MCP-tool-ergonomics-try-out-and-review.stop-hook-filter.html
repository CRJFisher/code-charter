The Stop-hook flow-relevance filter

The hook blocks the turn for any edited file — including docs/config that can never form a flow — so a doc-only turn triggers a full-repo reconcile that no-ops. This page shows the trigger chain, where one filter slots in, and the standalone-doc decision that falls out of it for free.

TL;DR — the decision you are confirming

Add one flow-relevance pre-filter to worked_on in the Stop-hook bin, just before decide_stop_action. A file is flow-relevant if is_supported_source(path) or find_skill_root(path, cwd) is defined. The existing worked_on.length === 0 guard then naturally no-ops an all-non-flow turn. AC#6 is option (a) — ignore-and-log: a standalone .md is simply dropped by the filter and never blocks; giving docs their own flow (option b) is deferred.

The root cause — the filter is missing, not the downstream guard

The reconcile engine already ignores non-flow files: reconcile()'s partition (lines 52–62) routes a file to a skill bundle only when find_skill_root hits, and to code only when is_supported_source holds — everything else is silently dropped. So a README.md reaching the reconciler is a guaranteed no-op.

The waste is upstream: the Stop hook still decides to block the turn and launch the reconciler sub-agent for that file, paying the full-repo, uncached Ariadne index before the engine drops it. The fix is to apply the same flow-relevance test the engine uses, one step earlier — in the hook, before decide_stop_action sees the list.

Two filters, two layers — not the same thing

The watermark (commit cb51e03, already shipped) scopes worked_on to this turn's edits — it answers "when." The flow-relevance filter (this task) drops the non-flow edits within that set — it answers "which." They compose: the new filter layers on top of the watermark, it does not replace it.

The insertion point in drift_stop_hook.ts

packages/drift/src/bin/drift_stop_hook.ts — main() (filter inserted)
  const { worked_on, next } = worked_on_since(transcript_text, payload.transcript_path, prev);

  // advance the cursor every fire …
  fs.writeFileSync(state_path, serialize_watermark(next));

  // AC#5: keep only files that can form a flow — a supported source extension, or under a SKILL.md.
  const flow_relevant = worked_on.filter(
    (f) => is_supported_source(f) || find_skill_root(to_abs(f, payload.cwd), payload.cwd) !== undefined,
  );

  const decision = decide_stop_action(payload, flow_relevant); // was: worked_on

Illustrative shape — is_supported_source and find_skill_root are imported from the reconcile module; to_abs resolves the repo-relative path against payload.cwd exactly as reconcile() does. The decision function itself is untouched.
