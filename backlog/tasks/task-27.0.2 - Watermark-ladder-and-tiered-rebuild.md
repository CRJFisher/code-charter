## Description

<!-- SECTION:DESCRIPTION:BEGIN -->

Preservation across regeneration — the load-bearing guarantee of the whole model. Implements the field-granularity watermark on the precedence ladder `user (2) > agentic (1) > raw (0)` and the per-tier rebuild, so a cheap, deterministic raw re-parse never clobbers an expensive agentic field, and an explicit agentic pass never clobbers an irreplaceable user edit.

The ladder is what lets the disposable raw tier be nuked and rebuilt at will while the preserved tiers above it survive untouched.

<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria

<!-- AC:BEGIN -->

- [ ] #1 `write_fields(target, fields, as_tier)` writes a field only if `TIER_RANK[current_owner] <= TIER_RANK[as_tier]`, stamps every written field as `as_tier` in `field_ownership`, and returns `{ skipped: string[] }` for the fields a higher tier owns; a field absent from `field_ownership` is treated as raw-owned (rank 0 per `TIER_RANK`)
- [ ] #2 A node `description` generated by an agentic pass is stamped agentic-owned (not left absent/raw-owned); a user edit then promotes it to user-owned, after which neither a raw re-parse nor an agentic pass overwrites it. Generally: a user-owned field survives both a raw re-parse and a later agentic pass unchanged; an agentic-owned field survives a raw re-parse but is overwritten by an agentic pass
- [ ] #3 `rebuild_layer('raw', write)` and `rebuild_layer('agentic', write)` nuke only that tier's rows plus any cache marked disposable in `table_disposition()` (e.g. `anchor_resolution`), run `write` in a transaction (task-27.0.1's primitive), and preserve every higher-tier row — and, because the writer goes through `write_fields`, every higher-tier-owned field on the rows it does rewrite (enforced by the AC#1 ladder, not by a re-check inside `rebuild_layer`)
- [ ] #4 The rebuild consults `table_disposition()` as data, never a hard-coded name list (verified by a spy/assertion in test)
- [ ] #5 `rebuild_layer` leaves a soft-deleted (`deleted_at` set, per task-27.0.1) agentic/user row's flag untouched — it neither hard-deletes the row nor clears the flag — so it remains restorable after the rebuild
- [ ] #6 Tests cover the full ladder matrix (raw/agentic/user × raw-pass/agentic-pass/user-edit) and the rebuild-preserves-higher-tiers property on a `:memory:` store

<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->

1. Implement `write_fields` as the single ladder-respecting write: per field, compare `TIER_RANK[current_owner]` against `TIER_RANK[as_tier]` (`json_extract`/`json_set` on `field_ownership` is the SQLite encoding of that engine-agnostic comparison); write the fields that pass, collect the rest into `skipped`.
2. Route every producer through it — raw re-parse writes `as_tier='raw'`, the agentic pass `'agentic'`, user edits `'user'`. The agentic pass stamps a generated `description` agentic-owned so a later user edit can promote it.
3. Implement `rebuild_layer(layer, write)`: in a transaction, delete the layer's rows plus any cache `table_disposition()` marks disposable (e.g. `anchor_resolution`), run `write`; higher tiers and their watermarked fields survive because the writer goes through `write_fields`.
4. Tests: ladder matrix, dual-sourced description promotion (raw-absent → agentic → user), rebuild-preserves-higher-tiers, soft-deleted survival, `table_disposition` consulted (spy).
