The crux

The existing stitch test hand-builds a synthetic CallGraph — it proves the plumbing but gives an executor nothing real to read. This harness instead commits three mini TypeScript codebases, parses them with real Ariadne, and asserts at the StitchBatch boundary: ground truth is authored into the fixtures (stitchable or not), so the same tests eval stub executors now and the live agent's judgment later.

The detection pipeline under test

The three fixture patterns

dispatch looks handlers up by key, so both its calls are unresolved (2/2 ≥ the 0.5 ratio over ≥2 sites) — it is the only candidate source; the pure handlers are targets.

2callback_wiring/

3unrelated_pair/
null → 2 singletons confirm → 2 singletons (false-positive guard)

Two genuinely independent domains: with zero unresolved shapes the batch is empty by construction, so even a confirm-everything stub cannot merge them.

What counts as an unresolved shape

Counts — parameter call, resolutions: []

callback_wiring/scheduler.ts (fixture)
export function schedule(before: () => void, after: () => void) {
  before();  // call ref, resolutions: [] → unresolved
  after();   // call ref, resolutions: [] → unresolved
}            // 2/2 unresolved · ≥2 sites → UnresolvedShape

Calling a function-typed parameter is an ordinary call reference whose resolution list is empty — Ariadne does not mark it is_callback_invocation — so it counts toward the unresolved ratio and produces the shape AC#2 asserts on.

Excluded — synthetic callback edge

gap_detection.ts · find_unresolved_shapes
for (const call of node.enclosed_calls) {
  if (call.is_callback_invocation) continue;
  // synthetic edges Ariadne adds for RESOLVED
  // callback patterns are not comprehension gaps
  call_site_count += 1; …

When Ariadne does resolve a callback pattern it adds a synthetic is_callback_invocation edge; the gate skips those, so fixtures must rely on genuinely unresolved sites, never on resolved callback plumbing.
