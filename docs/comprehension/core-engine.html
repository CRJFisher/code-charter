Core Engine

@code-charter/core module by module: the only place a diagram becomes durable — storage, model + flows, resolver, extractors, reextract, agentic — making no model calls.

package @code-charter/core deps types · @ariadnejs/types · graphology submodules 6 LLM injected, never called here

How the submodules compose

Six submodules form one pipeline from Ariadne's call graph to preserved, renderable rows; the field ladder is the trust floor under all of it.

raw / disposable substrate agentic / LLM lane deterministic engine store-mutating write loss / miss state olive→ = writes store · clay→ = reads/derives identity · grey→ = data hand-off · dashed = inferred

The LLM is an injected seam — core itself makes no model calls. Tiers split by cost of regeneration: only raw is nuked and rebuilt; agentic and user are carried across edits.

storage — the SQLite GraphStore

The single concrete persistence engine behind the engine-agnostic GraphStore contract; the three tiers are one layer column, and only anchor_resolution is disposable.

FileRole
storage/sqlite_graph_store.tsOnly file importing node:sqlite. Schema lifecycle, node/edge/provenance CRUD, soft-delete, write_fields (ladder + layer promotion), file-incidence reads, rebuild_layer.
storage/schema.tsDDL as SQL strings, CURRENT_SCHEMA_VERSION, TABLE_REGISTRY_SEED — per-table disposable/preserved disposition read as data.
storage/field_ladder.tsapply_field_ladder: single source of the per-field precedence ladder (user > agentic > raw); shared by store and model for identical accept/skip.
storage/null_graph_store.tsNullGraphStore — null-object degraded store for hosts below Node 22.13.0; reads empty, writes no-op.
index.tsopen_graph_store(db_path) — gates on host support, lazily require()s the engine only when supported.

Write at tier T lands iff owner ranks ≤ T; a user write promotes row.layer->user so rebuild_layer can't reach it. Schema mismatch DROPs only disposable tables, re-runs idempotent DDL. Implements GraphStore from packages/types/src/graph_store.ts.

model — graph model, flows & projection

CustomGraphModel is the single live, mutating surface over the store; flows induce members on demand and project to adapter-ready rows.

FileRole
model/custom_graph_model.tshydrate / mutate / flush over the store; the read-side render() fold of LayerSpec[] into a fresh graph; graph_to_rows for the UI boundary.
model/flow.tsFlow entity: build_skeleton_flows, reachable_from/induce_members, identity flow_id_of, order_flows, persisted reconstruction, paths_of, agentic.flow/agentic.flow_member builders.
model/flow_projection.tsproject_flow/project_hydrated_flow: induce members, synthesize render-only code.function nodes + code.calls edges, fold in scaffold, bounded by budget.
model/module_scaffold.tsbuild_module_scaffold: one agentic.group per defining file via a pluggable ModuleResolver, linked by agentic.contains edges.

Flow identity is the dominant seed's symbol_path via flow_id_of — content_hash excluded so a body edit never re-keys the flow. Membership is re-derived by induce_members, never stored; a shared leaf belongs to multiple flows.

Render budget DEFAULT_FLOW_BUDGET = {max_nodes:200, max_edges:400}. Under budget, leaves render nested in file-module groups; over budget the flow collapses to module granularity (lift_edges_to_modules), keeping whole-flow coverage at lower resolution rather than truncating.

resolver — anchoring

The single, pure, sqlite-free place a stored anchor becomes current code state; both diagram↔code directions share exactly this mechanism.

FileRole
resolver/resolve_anchor.tsFront door: resolve_anchor(anchor, index) runs the 4-arm cascade, returns a ResolveResult. Pure.
resolver/code_state.tsbuild_symbol_path, compute_content_hash (rename-stable), compute_span_hash, derive_code_state.
resolver/resolver_index.tsbuild_resolver_index: one CodeState per symbol, indexed by_symbol_path & by_content_hash; throws on duplicate symbol_path.
resolver/from_ariadne.tsAriadne seam: walks per-file defs, slices bodies from body_scope_id, bridges SymbolId↔symbol_path. Skips the <anonymous> sentinel.
resolver/anchor_string.tsformat_anchor/parse_anchor round-trip symbol_path:content_hash, splitting on the last colon (hash is 64 hex chars).

build_resolver_index throws on a duplicate symbol_path; from_ariadne skips the <anonymous> sentinel — both protect hit/body-changed from input-order dependence. symbol_path = file#enclosing.name:kind; content_hash strips the symbol's own identifier so a rename stays hash-stable. Consumes contracts from packages/types/src/graph_store.ts.

extractors — literal structure

Hand-rolled, dependency-free parsers turn skill bundles into raw-tier doc nodes and literal doc edges with click-through provenance — no inference, no LLM.

FileRole
extractors/skill_ingest.tsOrchestrator: ingest_skill(store, skill_dir, deps) walks a bundle, dedupes nodes/edges, does a scoped invalidate+upsert. Injects file IO via SkillIngestDeps so core imports no fs.
extractors/markdown_links.tsparse_markdown_links: inline [text](target) links, excluding fenced blocks & inline-code, with absolute source ranges.
extractors/frontmatter.tsparse_frontmatter: tolerant hand-rolled YAML for the leading --- block; surfaced only as node attributes.
extractors/meta_json.tsread_sub_agents: reads sub_agents[], locating each name literal's span. Shared with the agentic registry detector.
extractors/extractor_ids.tsCatalog: literal.markdown/literal.frontmatter/literal.meta_json ids and kind constants (code.doc, code.literal-doc).

Scoped invalidate+upsert per bundle, never the store-global rebuild_layer — so ingesting one skill never nukes another's raw rows; file IO is injected. The drift package's skill_dir.ts wraps ingest_skill with node:fs.

reextract — symbol-level change detection

The funnel that turns a turn's file edits into a precise symbol-granular SymbolDelta, so drift follows the code change, not the file save.

FileRole
reextract/re_extract.tsSingle entry point re_extract(file_set, origin, deps): invalidate+re-extract raw, rebuild file-module scaffold, resolve each preserved anchored node (re-anchor a relocation inline / soft-delete a miss), accumulate baseline, return SymbolDelta. Host supplies extract_raw/build_index.
reextract/symbol_delta.tscompute_symbol_delta(baseline, index): diffs fresh index vs persisted-anchor baseline into {added, removed, modified, relocated} by symbol_path. Pure.

added is relative to anchored knowledge and correct only when build_index is scoped to changed files alone; a whole-repo index would flood it. A relocated symbol is never re-described — the anchor, defining path and description side-node re-key inline, authored content byte-for-byte. Verdicts apply in two phases (all soft-deletes, then all re-key upserts) so same-pass rename chains land cleanly. Consumed by the drift package's reconcile.ts and affected_flows.ts.

agentic — descriptions, gaps, stitches & bridges

The agentic substrate deterministically proposes what static extraction cannot, then persists it on the agentic lane — making no model calls.

FileRole
agentic/gap_detection.tsdetect_gaps + derive_candidate_seeds: deterministic 3-category gap detector (orphan entrypoints, unresolved shapes, disconnected components) over the call graph; feeds the stitch seam. Pure.
agentic/stitch.tsbuild_candidate_stitches + null_stitch_executor: the EntrypointStitchExecutor injected seam; default confirms nothing (one-entrypoint-per-flow, deterministic). Consumed by drift/reconcile.
agentic/describe_policy.tsplan_descriptions: partitions members into cached / from_docstring / needs_llm / placeholder; defines injected DescribeBatchExecutor + null_describe_executor.
agentic/bridge.tsbuild_bridge_edges: collapses candidates into agentic.bridge EdgeRows (layer agentic, confidence 0.5, dashed render) with justifying provenance.
agentic/registry_detector.tsdetect_meta_json_sub_agent_bridges: reads meta.json sub_agents[], emits BridgeCandidates via an injected resolve_target seam.
agentic/write_descriptions.tsPersists descriptions as agentic.description side-nodes (one per symbol) via write_fields(...,'agentic'), honoring the ladder.
agentic/agentic_writer.tswrite_agentic_substrate: takes a SubstrateProposal (bridges + descriptions), persists both scoped, with count caps and a coarse deadline gate.

Descriptions are agentic.description side-nodes (anchored to symbol_path:content_hash) so a raw re-parse leaves them to re-anchor through re_extract; reads use include_deleted:true to avoid resurrecting soft-deleted rows. Sole real consumer is the drift hydration sub-agent, which fills the seams.
