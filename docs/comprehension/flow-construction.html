Flow Construction

A flow is v1's unit of comprehension — an agent-detected functionality umbrella over one or more call graphs. It is constructed in two tiers: a deterministic skeleton that is always available, and an agentic hydration that enriches it.

Where the agent runs

The drift-reconciler is Claude — a bounded sub-agent (tools: Skill, Bash) that runs in its own context so diagram work does not flood the main session. Its sole job: invoke the drift-sync skill, return one acknowledgement line. The skill shells into the headless drift-reconcile bin, which calls reconcile() with no executors injected. Both seams (stitch_entrypoints, describe) are optional fields in ReconcileDeps and are left undefined — they fall back to their null no-op implementations. In v1, no model token is spent on flow content. "Agentic" is the storage tier name, not a signal that a model generated the content.

The drift-reconciler is Claude but is constrained to context isolation only — it orchestrates the skill and nothing more. The two seams are TypeScript functions injected into ReconcileDeps; the bin is a separate spawnSync child and cannot receive function types across the process boundary. assets/agents/drift-reconciler.md · bin/drift_reconcile.ts

Two tiers of one flow

The deterministic skeleton is recomputed each session from the call graph and costs nothing to regenerate; the agentic hydration is an expensive, non-deterministic upgrade persisted once and preserved thereafter. The line between them is the cost-of-regeneration seam.

The skeleton is the permanent cheap substrate; hydration upgrades a skeleton stub in place to the same id, so the agentic layer enriches rather than replaces. core/model/flow.ts

The deterministic skeleton

build_skeleton_flows(graph) is the path the VS Code extension renders first, on a cold repo with no store. Each top-level entrypoint seeds one flow; forward-reachability fills its interior; the dominant seed's symbol_path names it; everything reachable from no entrypoint falls into the single unattributed bucket.

1
Seeds. build_skeleton_flows sorts graph.entry_points by id and walks them in that order, so output is byte-stable across runs. One SkeletonFlow per entrypoint. build_skeleton_flows
2
Reachable. reachable_from(seed, graph) forward-traverses resolved call edges, visiting children in sorted id order; cycles guard, external/unresolved targets skip. The reachable set is the flow's member_count. reachable_from
3
Identity. flow_id_of(node) = the entrypoint's enclosing-free symbol_path (entrypoints are top-level, so the enclosing scope is []). The label is the entrypoint's name. flow_id_of
4
Rank + dedup. Sort by member_count descending, id ascending to break ties; de-duplicate by id keeping the first (largest) — two same-named methods on different classes collapse to one v1 symbol_path.
5
Unattributed. Every node in no entrypoint's reachable set is gathered, sorted, into the single UNATTRIBUTED_FLOW_ID bucket, appended last — the catch-all for dead/unreachable code. UNATTRIBUTED_FLOW_LABEL

Membership is never a stored leaf set — induce_members re-derives it on demand, so a leaf reachable from two entrypoints legitimately belongs to both flows. core/model/flow.ts

Identity excludes the body A flow id is the dominant seed's symbol_path only — the content_hash is deliberately excluded. Editing an entrypoint's body never re-keys its flow; a rename moves the name segment, re-anchored by the resolver when the flow is persisted.

Agentic hydration & induction

Hydration upgrades a skeleton stub into a persisted agentic.flow along a five-stage deterministic engine spine. A model can speak in exactly two seams: the describe seam supplies member descriptions (stage 4) and the stitch seam supplies boundary decisions (stage 1) — merging orphan entrypoints that syntactic analysis leaves split. Both default to headless null executors; in v1 the entire pipeline is deterministic (see the runtime chain).

deterministic · default path describe seam (optional · model supplies text) stitch seam (always crossed · executor fills ConfirmedStitch[]) agentic-lane store write
1
Grouping — always stitch-aware. HYDRATE routes every turn through detect_and_stitch_code_umbrellas — there is no conditional branch. Orphan entrypoints enter the stitch seam (1b): confirmed groups become multi-seed umbrellas, unconfirmed/overflow orphans stay singletons. Non-orphan skeleton flows take detect_code_umbrellas — one flow per entrypoint, kept if its induced members touch a changed file. drift/reconcile/reconcile.ts
1b
Stitch seam. build_candidate_stitches runs detect_gaps, keeps orphan entrypoints in the changed neighbourhood, and pairs each orphan whose reachable tree holds ≥1 unresolved shape with every other neighbourhood orphan as (source, target) candidates, capped at MAX_STITCH_CANDIDATES = 50. The executor — deps.stitch_entrypoints ?? null_stitch_executor — returns ConfirmedStitch[], union-found into multi-seed CodeUmbrellas whose id is the alphabetically-first seed's symbol_path; bridges and rationale flow through hydrate_code_flow unchanged. Full anatomy: Entrypoint stitching. core/agentic/stitch.ts
2
Induce. induce_members re-derives the member set by forward reachability from the umbrella's seeds, plus any deterministic bridge endpoints — never a stored leaf set. core/model/flow.ts
3
Bridges = heuristics. build_bridge_edges and detect_meta_json_sub_agent_bridges infer cross-graph and skill sub-agent links as agentic.bridge edges at BRIDGE_CONFIDENCE_INFERRED = 0.5 (dashed render); stitch bridges join the same plumbing. A pure code flow writes bridges: []. core/agentic/bridge.ts · registry_detector.ts
4
Descriptions — the describe seam. plan_descriptions is deterministic-first: cache → docstring → optional batched LLM ≤ DEFAULT_DESCRIBE_CAP = 200 → name placeholder. A flow hydrates headlessly with no model call; this seam supplies text only. core/agentic/describe_policy.ts
5
Persist. write_agentic_substrate + write_flow write the agentic.flow node and substrate row-by-row through scoped write_fields('agentic'), stamping last_synced_at, never rebuild_layer. drift/reconcile/hydrate.ts · flow_store.ts

Both seams are null no-ops in v1 — see the runtime chain. core/agentic/stitch.ts · drift/reconcile/reconcile.ts

The stitch seam is always crossed — the default confirms nothing HYDRATE has no conditional branch: detect_and_stitch_code_umbrellas runs every turn and resolves the executor as deps.stitch_entrypoints ?? null_stitch_executor. In v1 the bin injects no live executor, so the null default runs — confirming nothing and keeping grouping byte-identical to one-entrypoint-per-flow. The seam is infrastructure; filling it with a live executor is a v2 concern.
The single most load-bearing write rule Persisting a flow must never call rebuild_layer('agentic') — that call is store-global and would delete every other flow's agentic content, defeating lazy per-flow hydration. Field writes go through write_fields(..., 'agentic'), so a user-promoted (renamed/pinned) flow node survives untouched.

Entrypoint stitching

Ariadne's call-graph extraction is syntactic: dynamic dispatch, registry lookups and callback wiring frequently fail to resolve, and each unresolved callee is promoted to its own top-level entrypoint — one functionality fragments into several flows. Entrypoint stitching is hydration stage 1's repair pass, run on every HYDRATE turn: Detect → Propose → Judge → Merge, with the judging step the only place a model speaks.

1
Detect. detect_gaps runs over the changed neighbourhood: orphan entrypoints are the candidate seeds; an UnresolvedShape flags a node when ≥50% of its ≥2 call sites carry no resolution (DEFAULT_GAP_OPTIONS). core/agentic/gap_detection.ts
2
Propose. build_candidate_stitches pairs each orphan whose reachable tree holds at least one UnresolvedShape — the evidence — with every other neighbourhood orphan as a StitchCandidate{source_seed, target_seed, unresolved_shapes_in_source}. Capped at MAX_STITCH_CANDIDATES = 50, truncation logged (never silent); overflow seeds remain in StitchBatch.neighbourhood_seeds as singleton fallbacks. core/agentic/stitch.ts
3
Judge. The executor resolves as deps.stitch_entrypoints ?? null_stitch_executor and returns ConfirmedStitch[] — label, inference_rationale, merged_seeds, and a justifying bridge{src_symbol_id, dst_symbol_id, source_file, source_range}. null_stitch_executor confirms nothing. core/agentic/stitch.ts
4
Merge. build_stitch_groups union-finds confirmed seeds — groups sharing any seed collapse into one. Each group becomes one multi-seed CodeUmbrella whose id is the alphabetically-first seed's symbol_path; build_bridge_edges writes the recovered edges at BRIDGE_CONFIDENCE_INFERRED = 0.5 with the call-site span as provenance; unconfirmed seeds fall back to singleton flows. drift/reconcile/reconcile.ts

Grouping is independent of bridge writes — a stitch whose bridge endpoints fail to resolve still merges its seeds; a malformed stitch resolving to fewer than two seeds is skipped. drift/reconcile/reconcile.ts

The seam is always crossed — the default confirms nothing HYDRATE has no stitch branch: detect_and_stitch_code_umbrellas runs every turn and resolves its executor as deps.stitch_entrypoints ?? null_stitch_executor. The null default returns no confirmations — the result is byte-identical to one-entrypoint-per-flow, and the headless drift-reconcile bin stays fully deterministic. The seam is infrastructure for v2.
Bridge endpoints are symbol_paths, never raw SymbolIds A ConfirmedStitch carries Ariadne SymbolIds; build_stitch_groups converts both endpoints to rename-stable symbol_paths before persistence — src via paths_of, dst via the target seed's id (already a symbol_path) — so persisted agentic.bridge edges live in the store's id space and survive re-parses.

Skeleton ↔ hydrated reconciliation

The selector merges two populations: persisted hydrated flows take precedence, the deterministic skeleton fills the rest. Membership re-induces from rename-stable seeds. A superseded flow (dominant seed gone or renamed) is retired by the seed-gone soft-delete in reconcile.

Re-sync re-induces a flow from its stored seeds — the id is unchanged by construction; there is no identity remap. A flow whose dominant seed no longer resolves is retired by the seed-gone soft-delete in resync_persisted_flow (the orphaned row is harmless); a renamed seed re-hydrates as a fresh flow, and a freshly hydrated umbrella retires persisted flows whose seeds it subsumes via retire_flows_subsumed_by. drift/reconcile/reconcile.ts
