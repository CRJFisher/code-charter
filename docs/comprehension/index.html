What Code Charter is

The diagram is a peer artifact above the code — authored, not regenerated: the code is the how, the diagram is the what and why, and it survives every code change.

A flow's membership is re-induced on demand, never a stored leaf set — so a shared leaf legitimately belongs to many flows. core/flow

The cost-of-regeneration seam (L0/L1)

One line splits cheap-deterministic-rebuild from expensive-non-deterministic-regenerate, and that line dictates rebuild-vs-preserve everywhere.

raw / disposable agentic / preserved user / preserved one layer column · ranks 0/1/2 — not separate tables watermark: user > agentic > raw

Agentic content is preserved because regenerating burns tokens and yields a different map — the trust floor. A landed user write promotes the row's layer to user, so no rebuild can destroy it. core/storage

The package map

Five packages, one strict one-way dependency graph: types is the zero-runtime contract, core implements it, and drift/ui/vscode consume it.

PackageWhat it isDepends on
@code-charter/types Shared type vocabulary: three-tier GraphStore contract and row shapes (NodeRow/EdgeRow/ProvenanceRow), Anchor/CodeState/ResolveResult resolver types, flow-keyed surface, CodeCharterBackend interface, call-graph wire (de)serialization. graph_store.ts is pure TS with zero imports. @ariadnejs/types

Makes no model calls — the LLM is an injected seam. types · @ariadnejs/types · graphology

Source code to rendered diagram

Source becomes a navigable diagram through a fixed pipeline, with a drift loop writing back scoped after every code-editing turn.

1
Source code. The repo's .ts/.py/.rs files.
2
Ariadne CallGraph over repo-relative paths. AriadneProjectManager / HeadlessProject
3
Three-tier store persists raw rows beside agentic + user tiers. upsert_node / write_fields
4
Induce + project. Skeleton on cold repo; re-induce members, bound to budget. build_skeleton_flows / project_flow
5
Map to React Flow (agentic.contains → parentId), ELK layout. custom_graph_to_react_flow
↺
Drift loop. Hook learns worked files; reconcile hydrates/re-syncs; scoped write-back. Stop hook

The loop preserves everything you authored, so the next render is honest. drift/reconcile
