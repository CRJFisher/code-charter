Drift Sync

After every agent turn, the affected flow's diagram first-hydrates or re-syncs automatically and off your attention, preserving everything you authored.

The problem & v1's answer

A diagram authored once goes stale; one you must refresh by hand nobody does — so v1 is pure auto-sync, scoped to the flows you work on.

Every flow-relevant turn always re-syncs or first-hydrates and always preserves edits. Auto-sync is the v1 honesty guarantee.

The end-to-end sequence

One trigger, one sub-agent, one skill, one deterministic engine; the Stop hook is the sole trigger and never reconciles inline.

1
Stop hook fires. Claude Code invokes the hook on the Stop event; the payload carries only a transcript path, not an edited-file list. bin/drift_stop_hook.ts
2
Watermark + filter. parse_worked_on_files scans the transcript JSONL for Edit/Write paths past a per-repo watermark, then filter_flow_relevant drops files that form no flow. hooks/transcript_parser.ts · hooks/stop_watermark.ts · reconcile/flow_relevance.ts
3
Block + instruct. decide_stop_action no-ops on the loop guard or an empty set; else emits decision:'block' to launch the reconciler. hooks/stop_decision.ts
4
Sub-agent launch. Bounded custom sub-agent (tools: Skill, Bash); returns one acknowledgement line so it never floods the session. assets/agents/drift-reconciler.md
5
Skill. Validates the pinned CLI contract --files/--store/--repo-root/[--json]/[--dry-run], locates the bin via sidecar, shells in forwarding flags + exit code. assets/skills/drift-sync · scripts/drift_sync.js
6
Engine. Opens the store, builds the headless call graph, calls reconcile(file_set, deps), emits one JSON outcome per flow. bin/drift_reconcile.ts · reconcile/reconcile.ts

The sub-agent writes the store only through the skill; the drift MCP surface is recovery and never reconciles.

The watermark is load-bearing Reconcile writes the store, never the transcript — so without a per-turn cursor, parse_worked_on_files over the append-only transcript returns the cumulative edited set and the hook re-fires it forever (an unsatisfiable loop). The watermark scopes "this turn's edits" precisely.

The reconcile pipeline

reconcile() runs a fixed sequence of deterministic stages, all sorted for byte-stable output, all writes scoped row-by-row.

A
Partition. Normalize to repo-relative, dedupe, drop empties/out-of-repo; files under a SKILL.md ancestor → skill bundle, Ariadne sources → code_files, rest dropped.
B
Refresh raw. re_extract(code_files,'code-change',deps) re-extracts raw tier + returns aggregate SymbolDelta; ingest_skill_dir writes literal doc nodes. core/src/reextract/re_extract.ts
C
Affected flows. affected_persisted_flows selects via body drift (a modified member's live id in induced membership) + membership drift (induced set ≠ stored anchor_set). reconcile/affected_flows.ts
D
Decision. Per flow, dispatch on EXISTS(agentic.flow node) — none → HYDRATE, exists → RE-SYNC.
E
Reconcile. HYDRATE chain; RE-SYNC re-induces, carries descriptions via resolver, re-describes only added ∪ modified. reconcile/reconcile.ts
F
Preserve. Scoped write_fields('agentic') stamping last_synced_at; miss → re-attachment bin, relocation → staged drift_*.

Writes are row-by-row upsert + stale-edge retirement honoring the agentic ladder, so a user-promoted flow node survives.

The single most load-bearing rule write_flow must never call rebuild_layer('agentic') — that call is store-global and would delete every other flow's agentic content, defeating lazy per-flow hydration.

Hydrate vs re-sync

Exactly two branches per flow, keyed on whether a persisted agentic.flow node exists; both always update the diagram, neither ever gates.

"Hydrated" means precisely EXISTS(agentic.flow). The skeleton is the permanent cheap substrate the agentic layer enriches lazily.

Symbol-level change detection

Drift sync is driven by the symbols that changed this turn, not the files touched — a SymbolDelta keyed by symbol_path.

ClassResolver verdictMeaning
modifiedbody-changed (same path, new hash)Body changed in place. Triggers re-sync of flows it reaches; re-described.
relocatedrelocated (same body, new path)Rename / cross-file move. Staged as drift_*, not re-described — description carries across.
removedmiss (resolves nowhere)Symbol is gone. Preserved content soft-deletes into the re-attachment bin.
addedfresh path absent from baselineNo persisted anchor. Harmless alone; the membership-diff trigger (not added) pulls a new member into a flow.

compute_symbol_delta diffs the fresh resolver index of changed files against the persisted-anchor baseline.

Cascade order is load-bearing resolve_anchor evaluates body-changed before relocated, so a body still present at its own path is never misreported as relocated. added is correct only with a changed-files-scoped index; a whole-repo index would flood it.

Edit preservation

Preservation is the trust floor — a cheap re-parse must never clobber an expensive description or an irreplaceable hand edit.

reanchor_node strips the drift_* staging and leaves the authored description byte-for-byte intact.

The drift MCP surface

Reconciliation is automatic and silent; a genuine miss whose authored description is stranded needs human recovery — the only job of the drift MCP.

ToolDoes
drift_listEnumerate the re-attachment bin — soft-deleted preserved content whose anchor missed; each carries the stranded description + ranked candidates[].
drift_nextReturn the oldest bin entry or null; a stateless resolve-next loop, the bin shrinks as entries resolve.
drift_resolvereanchor a staged relocation · reattach a bin entry (bare or onto a chosen target) · delete (keep as tombstone). Takes explicit kind:'node'|'edge'.

MCP is user-facing recovery only, never reconciles. reanchor and reattach read from two different sources. bin/drift_mcp.ts serves over stdio.

SessionStart banner & installer

The banner is the backstop for a declined reconcile; the installer wires every surface through one portable .claude tree, idempotently.

All surfaces resolve the same .code-charter/graph.db; a half-built package fails loudly rather than installing dangling hooks.

Why the skill is dependency-free drift_sync.js imports only Node builtins and never @code-charter/core — it runs from an installed .claude with no guaranteed node_modules. The reconcile bin is the only thing allowed to import core, which is exactly why it exists as a separate process with a pinned CLI contract.
