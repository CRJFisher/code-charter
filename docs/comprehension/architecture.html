Architecture & Data Model

The data-model spine: five packages, one three-tier graph in SQLite, a pure anchoring resolver, and one thread from source code to a rendered diagram.

packages 5 model 3 tiers · 1 layer column store node:sqlite spec task-27.0 substrate shipping
The one organizing seam Everything is governed by cost-of-regeneration. A free deterministic raw substrate (Ariadne call/import edges, the function→file→directory scaffold, literal doc edges) is disposable — nuked and rebuilt from code. The expensive non-deterministic agentic + user structure (flows, labels, inferred bridges, descriptions, pins, positions) is preserved — regenerating it burns LLM tokens and yields a different map.

The shared three-tier model (task-27.0)

Three tiers are one layer column on shared tables, split by cost-of-regeneration, with TIER_RANK = {raw:0, agentic:1, user:2}.

Open kinds / origins / render-layers / dispositions let 27.2 drop in with no schema migration. New kind + UI component registers via packages/ui/src/components/code_chart_area/chart_node_types.tsx.

The SQLite GraphStore: schema, registry, ladder

Rebuild policy is read as data from table_registry; only the resolver cache is disposable, so irreplaceable rows are never auto-migrated.

preserved table disposable store-mutating write slate = the ladder invariant · clay arrow = layer promotion

Drift is derived, not stored: re-extract(changed_files) ⊖ stored_graph via edge_provenance. Ladder in packages/core/src/storage/field_ladder.ts; engine is packages/core/src/storage/sqlite_graph_store.ts (only file importing node:sqlite); open_graph_store degrades to NullGraphStore below Node 22.13.

Load-bearing invariant — the data-loss fix Structural-layer promotion: a landed user-tier write promotes the row's structural layer to user (one-directional, never demotes). Because rebuild_layer deletes by layer, this stops a rebuild from hard-deleting a user-owned field sitting on an agentic/raw-layer row. The shared field-bag helper stays structural-column-free for store/model parity.

Anchoring & the resolver

Preserved content references code by an anchor, and the pure resolver reports a verdict — never mutates, never decides policy.

no-op verdict staged drift loss / soft-delete order load-bearing: body-changed BEFORE relocated

Relocated is staged, not re-anchored inline; reanchor_node rewrites only the anchor column, authored fields byte-for-byte intact. Resolver lives in packages/core/src/resolver; scoring in packages/core/src/resolver/rank_candidates.ts.

End-to-end data path: code to diagram

One thread runs through every package; a cheap deterministic re-parse writes only raw and is forbidden from clobbering descriptions or hand edits.

1
Code → CallGraph. Maps of SymbolId→CallableNode over repo-relative paths. packages/vscode/src/ariadne/project_manager.ts · packages/drift/src/reconcile/headless_project.ts
2
Re-extract → raw + SymbolDelta. re_extract(file_set, origin) reconciles anchors; compute_symbol_delta diffs into {added, removed, modified, relocated} — whitespace-only edit is a no-op. packages/core/src/reextract/re_extract.ts
3
Persist into 3-tier store. Raw via upsert_node/upsert_edge; agentic/user via ladder-respecting write_fields. SymbolDelta scopes which flows re-sync — drift follows the symbol, not the save.
4
Flow induction & projection. induce_members (seeds + bridges + linked docs); project_flow/project_hydrated_flow synthesize render nodes, fold the scaffold, bound by DEFAULT_FLOW_BUDGET. packages/core/src/model/flow.ts
5
render() folds layers. Ordered LayerSpec[] (raw→agentic→user→overlay) by list order; later layers win per key; tombstones folded; dropped-endpoint edges pruned. Open seam for overlays / whole-repo map.
6
RenderedRows → React Flow. custom_graph_to_react_flow resolves kind→RF type, folds agentic.contains→parentId; apply_hierarchical_layout runs ELK (LRU-cached); render with culling, search, nav, provenance. packages/ui/src/components/code_chart_area/custom_graph_to_react_flow.ts

This is a persistent model, never regenerate-on-demand — preservation of agentic+user is the hard part, and what makes the diagram an authored artifact rather than a snapshot.
